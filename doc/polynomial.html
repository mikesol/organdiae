Polynomials¶

This module exists to implement piecewise polynomials that eventually allow for convolution across buses. In doing so, it implements polynomials, impulses, and delta functions as well. Uses of these objects can be seen in the following documents: Quick Guide to Organdiae and Organdiae in Six Concepts.

class organdiae.core.polynomial.DeltaFunction(tuples=())¶

A representation of a piecewise polynomial given by [Polge73]. This allows for easy convolution. It is really not meant to be touched, and as such, the docstrings are very minimal.

class organdiae.core.polynomial.PiecewisePolynomial(polynomials=[], pbounds=[], impulses=[], iplacements=[])¶

Representation of a piecewise polynomial using polynomials and impulses. Behaves exactly like polynomials do. All piecewise polynomials are implicitly cropped at 0 on either end.

Example

>>> from organdiae import *
>>> p1 = Polynomial(3.,2.,0.,5.)
>>> p2 = Polynomial(0.,-8.,1.)
>>> p3 = Polynomial(-3.,0.,0.,2.)
>>> i1 = Impulse(2.0)
>>> i2 = Impulse(-1.2)
>>> pp1 = PiecewisePolynomial([p1,p2,p3],[-5.,4.,5.,6.],[],[])
>>> pp2 = PiecewisePolynomial([p2,p3],[2.,3.,8.],[i1,i2],[0.,4.])
>>> print pp1+pp2
POLYNOMIALS ::: (0,), None, -5.0 >> (3.0, 2.0, 0.0, 5.0), -5.0, 2.0 >> (3.0, -6.0, 1.0, 5.0), 2.0, 3.0 >> (0.0, 2.0, 0.0, 7.0), 3.0, 4.0 >> (-3.0, -8.0, 1.0, 2.0), 4.0, 5.0 >> (-6.0, 0.0, 0.0, 4.0), 5.0, 6.0 >> (-3.0, 0.0, 0.0, 2.0), 6.0, 8.0 >> (0,), 8.0, () ;;; IMPULSES ::: 2.0, 0.0 >> -1.2, 4.
>>> print pp1-pp2
POLYNOMIALS ::: (0,), None, -5.0 >> (3.0, 2.0, 0.0, 5.0), -5.0, 2.0 >> (3.0, 10.0, -1.0, 5.0), 2.0, 3.0 >> (6.0, 2.0, 0.0, 3.0), 3.0, 4.0 >> (-3.0, 8.0, -1.0, 2.0), 4.0, 5.0 >> (0.0, 0.0, 0.0, 0.0), 5.0, 6.0 >> (-3.0, 0.0, 0.0, 2.0), 6.0, 8.0 >> (0,), 8.0, () ;;; IMPULSES ::: -2.0, 0.0 >> 1.2, 4.
>>> print pp1*pp2
POLYNOMIALS ::: (0.0,), None, 2.0 >> (0.0, -24.0, -13.0, 2.0, -40.0, 5.0), 2.0, 3.0 >> (-9.0, -6.0, 0.0, -9.0, 4.0, 0.0, 10.0), 3.0, 4.0 >> (0.0, 24.0, -3.0, 0.0, -16.0, 2.0), 4.0, 5.0 >> (9.0, 0.0, 0.0, -12.0, 0.0, 0.0, 4.0), 5.0, 6.0 >> (0.0,), 6.0, () ;;; IMPULSES ::: 6.0, 0.0 >> -397.2, 4.

Note that multiplication, here, is different than convolution. Multiplication superimposes the two polynomials and multiplies their values. For convolution of PiecewisePolynomials, check out the DeltaFunction documentation. Exponentiation is not defined.

g0integrator(v, n=False)¶

Solver for piecewise polynomials that never have negative area. v is the area you want, and the return value will give you the x you need to get that area. VERY strange results otherwise...

>>> from organdiae import *
>>> p = PiecewisePolynomial([Polynomial(11.,2.,3.,-0.25),Polynomial(0.3,2.,2.),Polynomial(0.3,2.3)],[-1.,4.,5.,6.],[],[])
>>> p.integrate().evaluate(4.1)
123.18316666666665
>>> p.g0integrator(123.18316666666665)
4.0999999999999996

OK so it’s not perfect...but almost! Set n to if your v is 0-1 normalized.

class organdiae.core.polynomial.Polynomial(*args)¶

Polynomial that behaves like a polynomial should under addition, subtraction, and multiplication.

Example

>>> from organdiae import *
>>> p1 = Polynomial(3.,2.,0.,5.)
>>> p2 = Polynomial(0.,-8.,1.)
>>> print p1+p2
(3.0, -6.0, 1.0, 5.0)
>>> print p1-p2
(3.0, 10.0, -1.0, 5.0)
>>> print p1*p2
(0.0, -24.0, -13.0, 2.0, -40.0, 5.0)
>>> print p1**2
(9.0, 12.0, 4.0, 30.0, 20.0, 0.0, 25.0)
